# `configs/` — конфигурации проекта

| Файл | Кто читает | Назначение |
|------|-----------|------------|
| `rules.json` | `dynamic/monitor.py` через `--rules` | Полный набор правил безопасности: блокируемые процессы, сетевые порты/хосты, файловые пути/расширения, лимиты ресурсов, реакции на нарушения, настройки логирования. |
| `config.example.json` | `dynamic/isolated_run.py` | Параметры песочницы: `network`, `file_access`, `max_processes`. |
| `example_rules.json` | (справочный) | Снимок JSON, который может прийти из web-интерфейса `/set_rule`. Структура совпадает с тем, что принимает `RuleManager`. |

Все три файла — это просто JSON, редактируются руками или через UI
дашборда. Изменения применяются при следующем запуске соответствующего
компонента.
